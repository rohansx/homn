## Phase 3: User Story 1 — Rule-based allow/deny on a real Claude Code session (P1) 🎯 MVP

**Goal**: A destructive Bash call is denied by a Rhai rule; a benign read is allowed silently; an unmatched call falls through to TUI ask. Audit log records every decision.

**Independent Test**: Spec §"User Story 1 / Independent Test".

### Tests for User Story 1 (TDD — write FIRST, ensure they FAIL before implementing)

- [ ] T020 [P] [US1] In `homn-policy/tests/`: table-driven test fixture for `default.rhai` covering all 3 acceptance scenarios from US1. Expects loader + evaluator types that don't exist yet.
- [ ] T021 [P] [US1] In `homn-audit/tests/`: integration test that inserts a synthesized `DecisionRecord` and queries it back via `homn-audit::tail()`.
- [ ] T022 [P] [US1] In `tests/integration/`: end-to-end test using a "fake Claude Code" fixture that POSTs a `PermissionRequest` payload to the daemon socket and asserts the response shape per [contracts/hook-protocol.md](./contracts/hook-protocol.md).

### Implementation for User Story 1

- [ ] T023 [P] [US1] In `homn-policy/`: `Engine` wrapper around Rhai. Configures `set_max_operations`, `set_max_call_levels`, etc., per [policy-language.md](../../docs/technical/policy-language.md). Exposes `eval(rule_set, request) -> EvalOutcome`. Tests from T020 must pass.
- [ ] T024 [US1] In `homn-policy/`: `RuleSet` loader. Parses a `.rhai` file into ordered rules with their source location. Surfaces parse errors with file:line.
- [ ] T025 [US1] In `homn-policy/`: rule eval order — deny → ask → allow — with per-rule + per-call wall-clock budgets.
- [ ] T026 [P] [US1] In `homn-policy/`: rules-file watcher using `notify`. Atomic hot-reload (parse new file completely; swap in only on success).
- [ ] T027 [P] [US1] In `homn-audit/`: `tail`, `query_by_decision`, `query_by_session` functions. Each is a Tokio task talking to the writer over a channel.
- [ ] T028 [US1] In `homn-audit/`: single-writer task that batches inserts at 10 ms windows; respects WAL mode. Writer survives panics by restarting from a Tokio supervisor task.
- [ ] T029 [US1] In `homn-hook/`: `homn hook permission-request` subcommand. Reads stdin JSON, POSTs to daemon, writes hook return per [contracts/hook-protocol.md](./contracts/hook-protocol.md).
- [ ] T030 [US1] In `homn-daemon/`: wire up the request handler — receives `decisions.create`, runs `homn-policy::Engine::eval`, writes to `homn-audit`, returns the response. End-to-end test from T022 must pass.
- [ ] T031 [P] [US1] In `homn-tui/`: rendered `Ask` prompt using `ratatui`. Hotkeys: `a` allow, `d` deny, `A` always-allow, `D` always-deny, `s` show generated rule, `q` defer to Claude.
- [ ] T032 [US1] In `homn-daemon/`: `Ask` flow — emits `AskOpened` BusEvent, registers a watch for TUI subscribers, blocks the hook response up to `hook.timeout_ms` waiting for an answer.
- [ ] T033 [US1] In `homn-daemon/`: TUI subscriber autostart — when the daemon receives an `Ask` and no graphical surface is registered, spawn the TUI prompt in the calling session's terminal.

**Checkpoint**: P1 done. User can run `claude` with the hook installed, hit a destructive Bash call, see the daemon deny it, and view the audit log.

### Tests for User Story 3 (TDD)

- [ ] T051 [P] [US3] Snapshot test for the prompt-regex matcher across multiple Claude Code prompt formats (collected in `tests/fixtures/cc-prompts/`).
- [ ] T052 [P] [US3] Integration test: full `homn run` against a fake-claude binary that simulates a y/n prompt.

### Implementation for User Story 3

- [ ] T053 [US3] In `homn-hook/`: `Pty` module wrapping `portable-pty`. Spawns child, plumbs stdout to user, taps a copy of the stream.
- [ ] T054 [US3] Prompt-regex matcher with configurable pattern from `pty_wrapper.prompt_regex`.
- [ ] T055 [US3] Daemon-decision race: parallel task posts to socket; arrives within `deny_race_window_ms` → wrapper writes `y\n` or `n\n` to child stdin.
- [ ] T056 [US3] `homn run` subcommand in `homn-bin/` that wires `Pty` + decision race together. Passes through child exit code.
- [ ] T057 [P] [US3] R-001 spike: confirm SIGWINCH propagation on macOS (see [research.md](./research.md) R-001). Promote to a regression test.

**Checkpoint**: Users with the wrapper get deny enforcement regardless of #19298.
