## Summary

Phase 1 ships in 4 weeks calendar time; HN launch on Week 4. See [docs/phases/phase-1-policy.md](../../docs/phases/phase-1-policy.md) for the week-by-week breakdown.

## Technical Context

- **Language/Version**: Rust stable (1.83+ at time of writing); MSRV pinned per release.
- **Primary Dependencies**:
  - `tokio` — async runtime
  - `rusqlite` + `tokio-rusqlite` — audit + learning storage
  - `rhai` — embedded policy DSL
  - `clap` (derive) — CLI dispatch + subcommand handling
  - `ratatui` + `crossterm` — TUI prompt rendering
  - `portable-pty` — PTY-tap wrapper
  - `notify` — inotify/kqueue policy-file watcher
  - `rmcp` — MCP server (stdio + Streamable HTTP)
  - `serde` / `serde_json` — JSON-line RPC
- **Storage**: SQLite (WAL mode) at `$XDG_DATA_HOME/homn/audit.db` and `$XDG_DATA_HOME/homn/learning.db`. No external DB.
- **Testing**: `cargo test` for unit; `assert_cmd` + `predicates` for CLI integration; a fixture-based "fake Claude Code" harness for end-to-end (a tiny Rust binary that emulates the hook protocol).
- **Target Platform**: Linux (kernel ≥ 5.4) + macOS (≥ 13). Windows explicitly out of v1.
- **Project Type**: Single CLI + daemon (single binary, subcommand-driven). See [ADR-0002](../../docs/architecture/adr/0002-rust-rhai.md).
- **Performance Goals**: P95 deterministic decision latency ≤ 50 ms (rule eval + audit write); P95 `ask` total round-trip ≤ 2.5 s (assumes human present); 1000+ decisions/sec sustained throughput on commodity hardware.
- **Constraints**:
  - Per-rule wall-clock budget 50 ms (Rhai `set_max_operations`).
  - Per-call total budget 200 ms across all rule eval.
  - Hook response window 30 s (under Claude Code's hook timeout).
  - PTY-wrapper deny race window 200 ms (configurable in `homn.toml`).
- **Scale/Scope**: Single-user; ~10k decisions/day is the high-watermark for a power user; audit DB ~30 MB/month at that rate.

**Structure Decision**: Cargo workspace with library crates per responsibility, re-exported by `homn-bin`. Matches [architecture/overview.md](../../docs/architecture/overview.md). Single-binary install; subcommand dispatch via `clap` derive. The crate split is dictated by the layer boundaries defined in [architecture/overview.md](../../docs/architecture/overview.md) and the test-first invariants of Constitution Principle VI.

## Complexity Tracking

| Violation                                                  | Why Needed                                                                                          | Simpler Alternative Rejected Because                                                                       |
|------------------------------------------------------------|-----------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------|
| Two integration paths (hook + PTY wrapper)                 | Anthropic bug #19298: hook `deny` is ignored. PTY wrapper guarantees deny semantic.                  | Hook-only would silently lose denies. Wrapper-only would force users to change their `claude` invocation.  |
| Separate `learning.db` next to `audit.db`                  | Learning needs frequent reads + a TTL on rejected suggestions; tables in `audit.db` would muddy queries. | Same DB is technically possible but reduces query separability and complicates retention rules.            |
| Per-rule + per-call latency budgets (not just per-call)    | User-authored rules can be pathological; per-call budget alone allows a single rule to eat the budget. | Per-call only would let one bad regex starve the rest. Per-rule + per-call lets one slow rule fail and others continue. |
